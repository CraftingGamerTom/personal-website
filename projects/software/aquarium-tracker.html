Aquarium Data Tracker

The Problem

I wanted to create a thoughtful and creative Christmas gift for a family friend. About a year before this quick project the giftee lost many rare a cherished fish in a freak heater accident caused by a lack of monitoring and awareness. By the time the giftee realized the tank was too hot, it was too late. It was tragic.

Feature Requirement
Computer placement Concealed & Maintenance Free
Notifications iOS & Android
Current Temp Monitoring Webview

Solution

So for Christmas (just 3 days away at the time) I decided to construct a simple system monitoring the current temperature allowing the giftee to view the current temperature remotely and recieve notifications (Twilio) if the fish tank goes outside of a safe range for the fish. With this, the giftee will be able to prevent any futher fish casualties.

Most importantly we want to design a system that can be easily built upon as new sensors / motors could be requested later

Strategy is to use Raspberry Pi 3 B+ as our microcontroller which would have a temperature sensor soldered (not Sodder) to it for data collection, python to do the actual data collection and processing on the software side, data storage occuring on the Raspberry Pi on MongoDB and Spring Framework driving the web server.

For now, we just want to see the current Temperature, but by implementing a database we can later add graphs to our front end to call the REST API to show cool trends.

The code can be found on my Github.

Software

One major struggle I has was will Spring Boot. As it turns out, Raspberry Pi only supports MongoDB v1 (32-bit). This means that I could not use the latest Spring Boot Starter as a Parent in my maven POM right out of the box. I did a lot of fighting since I went ahead and wrote all my database accessor code for the new version of MongoDB without realizing this to be an issue. As a neared my expected completion I realized this was a very time consuming mistake. I was required to change a lot of code to accommodate the legacy version of MongoDB. I began doing this by downgrading Spring. This broke a lot of other functionality I was using for hosting webpages and using controllers for a REST API. Once I had everything with the REST API working on the latest Spring and the Mongo DB Accessors working on legacy code I needed to find a way to mix the two.

I needed to override the controlled versions of the libraries that controlled MongoDB in my POM and PRESTO, everything came together and functioned properly.

Hardware

Simple soldering job. There is probably a lot of well deserved hate for soldering directly to the board but this was my first Raspberry Pi project and I was under a time crunch and needed the system to be easy to install. I am open to suggestions as to how to properly solve this issue in a prettier way.
