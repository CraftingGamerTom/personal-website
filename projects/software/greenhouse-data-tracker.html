The Problem

Quinsigamond Community College in Worcester, MA started a new greenhouse on campus and wanted a way to collect data on different aspects of the greenhouse. The goal was to automate processes, like cooling and heating based on the collected data like temperature, in the greenhouse. Members wanted to monitor the greenhouse data over the internet.

Type Requirement
User Accounts Required
Scaleable Sensor Types Yes
Secure Sign-In Yes

Solution

I created a web application that allowed users to record 'observations', which is anything that was noticed in the greenhouse and was worth noting. I also worked with another student, Brendan Russel to be able to monitor the data live and also keep a history of the data for future understanding. Brendans job was to automate the systems and deal with hardware (sensors). My job was to handle the data and create the front end for users to access data to the greenhouse.

I wrote a web application on top of the Spring Framework using Google OpenId with OAuth2 so that I did not need to manage user's passwords or any sensistive information.

This project eventually migrated to a private repository. I continue to license Quinsigamond Community College versions of the application and work with them closer to provide a better product for the greenhouse.

Data Collection

REST API for collecting data. This secure service handles json sent by a device containing sensor data. It is parsed and put into a database for future use.

Web Application (UI)

A web application written on the Spring Framework utilizes server-side scripting to generate useful UI for viewers to be able to monitor sensor data and plants.

Database (MongoDB)

I chose MongoDB to serve as the Database for the web application as it is very flexible. When I started, plans were very fluid and data needed to be able to adapt quickly. MongoDB has served as an amazing tool throughout this project. I must say, if I were to do this again I would use a SQL database rather than a non-SQL database such as MongoDB. Future work for this has been continued in a seperate project and has been converted over to a SQL structure.
